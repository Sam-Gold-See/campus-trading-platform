# 交易状态闭环与双向评价 - 后端技术设计文档

## 1. 概述

标记成交形成交易闭环，双方互评建立信用档案。


## 2. 接口

### 2.1 标记成交 `PUT /api/item/{id}/sold`

| 请求体 | 说明 |
|--------|------|
| matchedUserId | 成交对象ID |

校验：帖文存在 + 展示中 + 归属权 + 不能自成交 + 成交对象存在 → 更新 `item_status=1, matched_user_id=?`


### 2.2 提交评价 `POST /api/evaluate/submit`

| 字段 | 说明 |
|------|------|
| itemId | 关联帖文 |
| revieweeId | 被评价人 |
| ratingType | 1=好评, 0=中评, -1=差评 |
| tags | `List<String>` 标签 |
| content | 文字短评（≤50字） |

校验：帖文存在且已成交 + 双方参与该交易 + 不能自评 + 不能重复评。

### 2.3 评价列表 `GET /api/evaluate/user/{revieweeId}`

分页查询某用户收到的评价，含评价人昵称。返回 `List<ReviewResp>`。

### 2.4 检查是否已评 `GET /api/evaluate/check?itemId=&reviewerId=`

## 3. 评价标签库

| 好评 | 差评 |
|------|------|
| 爽快、准时、描述相符、态度好、信誉好 | 迟到、描述不符、态度差、砍价、失信 |
