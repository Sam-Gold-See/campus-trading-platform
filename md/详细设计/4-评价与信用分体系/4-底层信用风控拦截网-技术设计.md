# 底层信用风控拦截网 - 后端技术设计文档

## 1. 概述

对信用分低于阈值的用户强制权限剥夺，拦截发布帖子。

## 2. 实现方式：Spring Interceptor

`CreditInterceptor` 实现 `HandlerInterceptor`，注册于 `WebConfig`：

```java
registry.addInterceptor(creditInterceptor)
    .addPathPatterns("/api/item/publish");
```


## 3. 拦截逻辑

1. 从 `SecurityContextHolder` 取 userId
2. Redis 查 `user:credit:{userId}`（缓存 1 小时）
3. 缓存未命中则查 DB
4. 信用分 < `MIN_THRESHOLD`（60） → 403

## 4. 风控规则

| 规则 | 条件 | 限制 | 实现状态 |
|------|------|------|:--:|
| R1 | <60分 | 禁止发布帖文 | ✅ |

## 5. CreditConstant

```java
BASE_SCORE = 100
MIN_THRESHOLD = 60
GOOD_REVIEW_SCORE = 2
BAD_REVIEW_SCORE = -2
DEAL_SCORE = 1
```
