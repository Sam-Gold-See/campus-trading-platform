# 校园用户认证与管理系统 - 后端技术设计文档

## 1. 概述

本文档详细描述校园交易平台用户认证与管理系统的后端技术实现方案，包括用户注册、登录、资料管理和登出等功能模块的技术架构、接口设计、数据模型和安全策略。

## 2. 技术栈

- **框架**: Spring Boot 4.x
- **安全**: Spring Security + JWT
- **数据库**: MySQL 8.0+
- **缓存**: Redis
- **密码加密**: BCrypt
- **验证**: Jakarta Validation

## 3. 核心功能模块设计

### 3.1 用户注册模块 (`/api/user/register`)

#### 3.1.1 API 设计

##### 发送注册验证码
- **接口**: `POST /api/user/sendRegisterCode`
- **请求参数**:
```json
{
  "eduEmail": "string" // 教育邮箱，必须符合 *.edu.cn 格式
}
```
- **响应参数**:
```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

##### 注册提交
- **接口**: `POST /api/user/register`
- **请求参数**:
```json
{
  "eduEmail": "string",      // 教育邮箱
  "verificationCode": "string", // 验证码
  "nickname": "string",       // 昵称 (4-16位，禁特殊字符)
  "password": "string",       // 密码 (8-20位，需包含字母和数字)
  "confirmPassword": "string" // 确认密码
}
```
- **响应参数**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "long",
    "nickname": "string",
    "token": "string",
    "accessExpire": "long"
  }
}
```

#### 3.1.2 业务逻辑设计

##### 验证码服务
- **Redis Key 格式**: `register:email:{email}` -> `{code}`
- **有效期**: 5分钟
- **限流策略**: 同一邮箱60秒内只能发送一次验证码

##### 注册流程
1. 校验邮箱格式（@*.edu.cn）
2. 验证验证码有效性
3. 校验邮箱是否已注册
4. 校验昵称唯一性
5. 校验密码强度（8-20位，含字母和数字）
6. 使用BCrypt对密码进行加盐哈希
7. 初始化用户信息（credit_score=100, status="active"）
8. 保存用户到数据库
9. 生成JWT Token并返回

### 3.2 用户登录模块 (`/api/user/login`)

#### 3.2.1 API 设计

##### 登录认证
- **接口**: `POST /api/user/login`
- **请求参数**:
```json
{
  "eduEmail": "string",     // 教育邮箱
  "password": "string",     // 密码
}
```
- **响应参数**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "long",
    "nickname": "string",
    "token": "string",
    "accessExpire": "long"
  }
}
```

#### 3.2.2 业务逻辑设计

##### 登录策略分发
- 如果提供 `password`，执行密码验证
- 如果提供 `verificationCode`，执行验证码验证

##### 安全防御机制
- **失败计数**: Redis记录登录失败次数，key格式：`login:fail:{email}`
- **账号锁定**: 15分钟内连续5次失败，锁定账号30分钟，key格式：`login:lock:{email}`

##### Token机制
- 目前仅实现Access Token（1天过期）
- 预留Refresh Token机制扩展空间

### 3.3 账户资料管理模块 (`/api/user/profile`)

#### 3.3.1 API 设计

##### 获取用户资料
- **接口**: `GET /api/user/profile`
- **请求头**: `Authorization: Bearer {token}`
- **响应参数**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "long",
    "eduEmail": "string",
    "nickname": "string",
    "avatarUrl": "string",
    "creditScore": "integer",
    "userStatus": "integer",
    "createdAt": "datetime"
  }
}
```

##### 修改昵称
- **接口**: `PUT /api/user/editNickname`
- **请求头**: `Authorization: Bearer {token}`
- **请求参数**:
```json
{
  "nickname": "string" // 新昵称
}
```

##### 修改密码
- **接口**: `PUT /api/user/editPassword`
- **请求头**: `Authorization: Bearer {token}`
- **请求参数**:
```json
{
  "oldPassword": "string",    // 旧密码
  "newPassword": "string",    // 新密码
  "confirmNewPassword": "string" // 确认新密码
}
```

#### 3.3.2 业务逻辑设计

##### 昵称修改限制
- **唯一性校验**: 检查新昵称是否已被其他用户使用
- **冷却期**: 每30天仅能修改一次，Redis记录上次修改时间
- **内容审计**: 集成敏感词库过滤违规昵称

##### 密码修改安全
- **身份验证**: 从JWT解析当前用户ID
- **旧密码验证**: 必须正确输入旧密码
- **强制下线**: 改密成功后使该用户其他设备的Token失效

### 3.4 登出与认证失效模块 (`/api/user/logout`)

#### 3.4.1 API 设计

##### 登出销毁
- **接口**: `POST /api/user/logout`
- **请求头**: `Authorization: Bearer {token}`
- **响应参数**:
```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

#### 3.4.2 业务逻辑设计

##### Token黑名单机制
- 将当前Access Token加入Redis黑名单
- Key格式：`token:blacklist:{token}`
- 有效期：Token剩余过期时间
- 在JWT过滤器中检查Token是否在黑名单中

## 4. 数据模型设计

### 4.1 用户表 (users)

```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    edu_email VARCHAR(100) UNIQUE NOT NULL COMMENT '教育邮箱',
    password_hash VARCHAR(255) NOT NULL COMMENT '密码哈希值',
    nickname VARCHAR(50) UNIQUE NOT NULL COMMENT '昵称',
    avatar_url VARCHAR(255) DEFAULT NULL COMMENT '头像URL',
    credit_score INT DEFAULT 100 COMMENT '信用分',
    user_status TINYINT DEFAULT 0 COMMENT '账号状态: 0正常, 1禁用',
    is_admin TINYINT DEFAULT 0 COMMENT '是否管理员: 0普通用户, 1管理员',
    last_nickname_change DATETIME DEFAULT NULL COMMENT '上次昵称修改时间',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    
    INDEX idx_edu_email (edu_email),
    INDEX idx_nickname (nickname)
);
```

### 4.2 Redis数据结构设计

| Key格式 | 类型 | 用途 | 过期时间 |
|---------|------|------|----------|
| `register:email:{email}` | String | 注册验证码 | 5分钟 |
| `login:fail:{email}` | String | 登录失败计数 | 15分钟 |
| `login:lock:{email}` | String | 账号锁定标记 | 30分钟 |
| `nickname:cooldown:{userId}` | String | 昵称修改冷却 | 30天 |
| `token:blacklist:{token}` | String | Token黑名单 | Token剩余时间 |

## 5. 安全设计

### 5.1 密码安全
- 使用BCrypt算法进行密码哈希
- 每次哈希使用不同的salt
- 不在日志中记录明文密码

### 5.2 JWT安全
- 使用HS256签名算法
- Secret密钥长度不少于32字节
- Access Token有效期1天
- 实现Token黑名单机制

### 5.3 防暴力破解
- 登录失败计数机制
- 图形验证码防护
- 账号临时锁定机制

### 5.4 输入验证
- 邮箱格式正则校验（*.edu.cn）
- 昵称长度和字符限制
- 密码强度要求
- SQL注入防护（MyBatis参数化查询）

## 9. 部署配置

### 9.1 环境变量配置
```yaml
# application.yaml
ctp:
  root:
    edu-email: ${ROOT_EDU_EMAIL:admin@university.edu.cn}
    password: ${ROOT_PASSWORD:defaultPassword123}

jwt:
  secret: ${JWT_SECRET:your-secret-key-must-be-at-least-32-characters-long}
  expire: ${JWT_EXPIRE:86400} # 1 day in seconds

spring:
  redis:
    host: ${REDIS_HOST:localhost}
    port: ${REDIS_PORT:6379}
    password: ${REDIS_PASSWORD:}
```

```yaml
spring:

    lettuce:
      pool:
        max-active: 8
        max-idle: 8
        min-idle: 0
        max-wait: -1ms
```
