# 校园用户认证与管理系统 - 后端技术设计文档

## 1. 概述

本文档描述校园交易平台用户认证与管理系统的后端技术实现，包括注册、登录、资料管理、登出。


## 2. 技术栈

- **框架**: Spring Boot 4.x
- **安全**: Spring Security + JWT
- **数据库**: MySQL 8.0+
- **缓存**: Redis
- **密码加密**: BCrypt

## 3. 核心功能

### 3.1 用户注册

#### API
| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/user/sendRegisterCode` | POST | 发送注册验证码 |
| `/api/user/register` | POST | 注册提交 |

#### 逻辑
1. 校验邮箱格式 `@*.edu.cn`
2. 60 秒内同一邮箱不可重复发送验证码（Redis: `register:email:rate:{email}`）
3. 验证码 5 分钟有效（Redis: `register:email:{email}`）
4. BCrypt 密码哈希、昵称唯一性校验
5. 初始信用分 = 100，user_status = 0

### 3.2 用户登录

#### API
| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/user/login` | POST | 邮箱+密码登录 |


#### 逻辑
1. 查用户 → 验密码 → 签发 JWT（HS256, 1天过期）

### 3.3 资料管理

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/user/profile` | GET | 查本人完整信息（含未脱敏邮箱） |
| `/api/user/profile/{userId}` | GET | 查他人公开信息（邮箱脱敏 `t***6@m.scnu.edu.cn`） |
| `/api/user/editNickname` | PUT | 修改昵称（30天冷却期） |
| `/api/user/editPassword` | PUT | 修改密码（需旧密码） |


### 3.4 登出

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/user/logout` | POST | 当前 Token 加入 Redis 黑名单 |

JWT 过滤器检查黑名单（`token:blacklist:{token}`），命中则 401。

## 4. 数据模型

### user 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | |
| edu_email | VARCHAR(100) UNIQUE | 教育邮箱 |
| password_hash | VARCHAR(255) | BCrypt 密文 |
| nickname | VARCHAR(50) UNIQUE | 昵称 |
| avatar_url | VARCHAR(255) | 头像 |
| credit_score | INT DEFAULT 100 | 信用分 |
| user_status | TINYINT DEFAULT 0 | 0=正常, 1=禁用 |
| is_admin | TINYINT DEFAULT 0 | 0=用户, 1=管理员 |
| last_nickname_change | DATETIME | 上次改昵称时间 |
| created_at | DATETIME | |
| updated_at | DATETIME | |

### Redis 键

| Key | 用途 | TTL |
|-----|------|-----|
| `register:email:{email}` | 验证码 | 5min |
| `register:email:rate:{email}` | 发送限流 | 60s |
| `nickname:cooldown:{userId}` | 昵称冷却 | 30天 |
| `token:blacklist:{token}` | 登出黑名单 | Token 剩余时间 |

## 5. 安全

- BCrypt 密码哈希，每次不同 salt
- JWT HS256 签名，密钥 ≥32 字节
- Token 黑名单（Redis）
- edu.cn 邮箱正则校验
- 昵称 4-16 位、密码 8-20 位含字母+数字
