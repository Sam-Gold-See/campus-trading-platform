# 我的发布与状态管理 - 后端技术设计文档

## 1. 概述

管理用户发布的帖文生命周期：列表查询（按状态筛选）、擦亮、下架、标记成交。


## 2. 接口

### 2.1 我的发布列表 `GET /api/item/my-list`

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| status | int | 0 | 0=进行中, 1=已完成, 2=已失效 |
| page | int | 1 | |
| size | int | 10 | |

返回 `List<ItemResp>`（不含分页元数据 total/page/size）。


### 2.2 擦亮 `PUT /api/item/{id}/bump`

校验归属权 → 更新 `created_at = NOW()`。


### 2.3 下架 `PUT /api/item/{id}/offline`

校验归属权 → 更新 `item_status = 2`。

### 2.4 标记成交 `PUT /api/item/{id}/sold`

| 请求体 | 说明 |
|--------|------|
| matchedUserId | 成交对象ID |

校验归属权 + 不能自成交 + 用户存在 → 更新 `item_status=1, matched_user_id=?`。


## 3. ItemResp DTO

```java
ItemResp {
    id, userId, userNickname, userCreditScore,
    type, categoryId, categoryName,
    campus, price, content, imageUrl,
    itemStatus, createdAt, expireAt
}
```

## 4. 状态流转

```
0(展示中) → 1(已成交) / 2(已失效)
```
