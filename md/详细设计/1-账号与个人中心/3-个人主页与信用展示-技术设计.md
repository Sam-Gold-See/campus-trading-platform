# 个人主页与信用展示 - 后端技术设计文档

## 1. 概述

提供用户个人资料查看功能，分为"本人完整信息"和"他人公开信息"两种模式。评价列表已迁移至模块5（`/api/evaluate/user/{revieweeId}`）。

## 2. 接口

### 2.1 获取本人资料 `GET /api/user/profile`

从 JWT 取 userId → 查 user 表 → 返回完整字段（含未脱敏邮箱、isAdmin）。

### 2.2 获取他人公开资料 `GET /api/user/profile/{userId}`

查 user 表 → 邮箱脱敏（`t***6@m.scnu.edu.cn`） → 不返回 isAdmin。

### 2.3 评价列表（模块5实现）`GET /api/evaluate/user/{revieweeId}`

由 ReviewController 提供，不在本模块。

## 3. 响应 DTO

```java
UserProfileResp {
    id, eduEmail, nickname, avatarUrl,
    creditScore, userStatus, isAdmin, createdAt
}
```

- 本人 `/profile` → 全部字段
- 他人 `/profile/{id}` → 无 isAdmin，邮箱脱敏

## 4. 评价表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | |
| item_id | BIGINT | 关联帖文 |
| reviewer_id | BIGINT | 评价人 |
| reviewee_id | BIGINT | 被评价人 |
| rating_type | TINYINT | 1=好评, 0=中评, -1=差评 |
| tags | VARCHAR(100) | 标签（逗号分隔） |
| content | VARCHAR(255) | 文字短评 |
| created_at | DATETIME | |
