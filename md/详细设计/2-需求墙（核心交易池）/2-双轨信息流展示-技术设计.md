# 双轨信息流展示（首页信息池）- 后端技术设计文档

## 1. 概述

首页按"求购"/"转让"双 Tab 展示信息卡片列表，支持滚动分页。

## 2. 接口 `GET /api/item/feed`

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:--:|--------|------|
| type | int | 否 | — | 1=求购, 2=转让，不传返回全部 |
| page | int | 否 | 1 | |
| size | int | 否 | 10 | |

返回 `List<ItemResp>`。

## 3. 查询逻辑

```sql
SELECT i.*, u.nickname, u.credit_score, c.name AS category_name
FROM item i
JOIN user u ON i.user_id = u.id
LEFT JOIN category c ON i.category_id = c.id
WHERE i.item_status = 0 AND i.expire_at > NOW()
  AND (type 条件可选)
ORDER BY i.created_at DESC
LIMIT offset, size
```

- 数据防腐：后端强制 `status=0 AND expire_at > NOW()`
- 排序：`created_at DESC`（擦亮过的自然置顶）
- 联表：user（昵称+信用分）、category（分类名）

## 4. ItemResp 返回字段

```java
id, userId, userNickname, userCreditScore,
type, categoryId, categoryName,
campus, price, content, imageUrl,
itemStatus, createdAt, expireAt
```
