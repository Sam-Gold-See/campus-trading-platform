# 克制化发布引擎 - 后端技术设计文档

## 1. 概述

本文档详细描述校园交易平台克制化发布引擎的后端技术实现方案，包括结构化配置字典获取、单图上传与压缩、帖文发布与内容审核等功能模块的技术架构、接口设计、数据模型和安全策略。

**核心设计原则**：降低发布门槛，强制信息结构化，净化内容生态。通过字数限制、图片限制、必填打标、违规词过滤等手段确保帖子质量。

## 2. 技术栈

- **框架**: Spring Boot 4.x
- **安全**: Spring Security + JWT
- **数据库**: MySQL 8.0+
- **缓存**: Redis
- **图片处理**: Thumbnailator（Java 缩略图库）
- **验证**: Jakarta Validation

## 3. 核心功能模块设计

### 3.1 结构化配置字典模块 (`/api/config`)

**业务目标**：统一返回系统维护的分类列表，供前端发布表单渲染下拉选择器。

#### 3.1.1 API 设计

##### 获取分类字典
- **接口**: `GET /api/config/dict`
- **请求头**: `Authorization: Bearer {token}`
- **响应参数**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "categories": [
      { "id": 1, "name": "书籍教材", "sortOrder": 1 },
      { "id": 2, "name": "二手数码", "sortOrder": 2 },
      { "id": 3, "name": "生活用品", "sortOrder": 3 },
      { "id": 4, "name": "服饰鞋包", "sortOrder": 4 },
      { "id": 5, "name": "校园代步", "sortOrder": 5 },
      { "id": 6, "name": "其他", "sortOrder": 6 }
    ]
  }
}
```

**说明**：
- 校区列表和价格区间由前端硬编码维护，减少接口复杂度
- 校区枚举：南校区、北校区、东校区、西校区、主校区
- 价格区间枚举：50元以下、50-200元、200-500元、500元以上、面议

#### 3.1.2 业务逻辑设计

##### 查询流程
1. 查询 `category` 表全量数据
2. 按 `sort_order` 升序排列
3. 封装返回 `List<CategoryResp>`

##### 缓存策略
- 分类数据极少变更，启动时加载到 Redis，TTL = 1 小时
- 接口优先读 Redis 缓存，缓存未命中则查询数据库并回写缓存

### 3.2 单图上传模块 (`/api/upload`)

**业务目标**：接收用户上传的补充实物图，服务端强压缩为缩略图（限制1080px / WebP 格式），限制最多 1 张。

#### 3.2.1 API 设计

##### 单图上传
- **接口**: `POST /api/upload/image`
- **请求头**: `Authorization: Bearer {token}`
- **请求方式**: `multipart/form-data`
- **请求参数**:
```
file: 图片文件（限 jpg / jpeg / png / webp，最大 5MB）
```
- **响应参数**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "imageUrl": "string"  // 压缩后的图片访问URL
  }
}
```

#### 3.2.2 业务逻辑设计

##### 上传处理流程
1. **文件类型校验**：检查文件扩展名和 MIME 类型，仅允许 `jpg / jpeg / png / webp`
2. **文件大小校验**：限制 5MB 以内，超限返回错误
3. **上传频率限制**：Redis 记录 `upload:limit:{userId}` = "1"，TTL = 60 秒，防止恶意高频上传
4. **图片压缩**：
   - 使用 `Thumbnailator` 库处理
   - 缩放至宽度 1080px 以内，保持原始宽高比
   - 输出格式统一为 WebP（相比原图节省 60-80% 带宽）
   - 压缩质量 0.8（平衡画质与文件大小）
5. **存储策略**：
   - 本地磁盘存储：`/uploads/images/{yyyyMM}/{uuid}.webp`
   - 配置 Spring MVC 静态资源映射，使图片可通过 URL 直接访问
6. **返回结果**：返回可访问的图片 URL 路径

##### 图片压缩参数

| 参数 | 值 | 说明 |
|:-----|:---|:-----|
| 最大宽度 | 1080px | 校园场景够用 |
| 输出格式 | WebP | 高压缩比 |
| 压缩质量 | 0.8 | 画质与大小平衡 |
| 保持比例 | true | 不变形 |

### 3.3 帖文发布模块 (`/api/item`)

**业务目标**：接收用户帖文，强制执行内容约束和结构化打标，写入 item 表并自动计算生命周期。

#### 3.3.1 API 设计

##### 提交发布申请
- **接口**: `POST /api/item/publish`
- **请求头**: `Authorization: Bearer {token}`
- **请求参数**:
```json
{
  "type": "integer",        // 交易类型：1=求购, 2=转让（必填）
  "categoryId": "integer",  // 分类ID（必填）
  "campus": "string",       // 校区/楼栋（必填）
  "content": "string",      // 文本描述，最长200字（必填）
  "imageUrl": "string",     // 图片URL（可选，来自上传接口返回）
  "price": "decimal"        // 期望价格（可选，不传或null代表面议）
}
```
- **响应参数**:
```json
{
  "code": 200,
  "message": "发布成功",
  "data": {
    "id": "long",           // 新帖文ID
    "createdAt": "datetime" // 发布时间
  }
}
```

#### 3.3.2 业务逻辑设计

##### 发布流程（有序）
1. **风控前置拦截**：
   - 查询当前用户信用分，若 `< 60` 分，直接拒绝发布
   - 错误信息："您的信用分低于60分，当前已被限制发布权限，请规范交易行为"
2. **参数校验**：
   - `type` 必须为 1 或 2
   - `categoryId` 对应的分类必须存在于 `category` 表
   - `campus` 非空
   - `content` 非空、不超过 200 字
3. **文本合规校验**：
   - 接入违规词库，使用 Trie 树算法匹配
   - 若包含违禁词，拦截并返回"内容包含违规信息，请修改后重新发布"
   - 违规词库从配置文件或 Redis 加载，支持热更新
4. **图片校验**：
   - `imageUrl` 非空时，校验 URL 格式合法性
   - 限制每帖最多 1 张图（前端已限制，后端二次校验）
5. **入库处理**：
   - `user_id` = 当前登录用户 ID（从 JWT 解析）
   - `item_status` = 0（展示中）
   - `created_at` = 当前时间
   - `expire_at` = 当前时间 + 14 天（系统自动计算）
6. **返回结果**：返回帖文 ID 和发布时间

##### 违规词库设计

| 类别 | 示例 | 存储方式 |
|:-----|:-----|:---------|
| 广告引流 | 加微信、扫码、关注公众号、QQ群 | Trie 树加载 |
| 色情低俗 | 约炮、一夜情、包夜 | Trie 树加载 |
| 违法交易 | 代考、代写、代课、刷单 | Trie 树加载 |
| 政治敏感 | 按网信办要求动态维护 | Redis 配置 |

**实现方式**：
- 启动时从配置文件 `sensitive-words.txt` 加载到 Trie 树内存结构
- 提供 Redis 缓存层，支持在不重启的情况下热更新词库
- 匹配时忽略大小写和常见变体（如全角/半角、空格）

##### 生命周期赋值
```java
LocalDateTime expireAt = LocalDateTime.now().plusDays(14);
item.setExpireAt(expireAt);
```

### 4.2 需求/物品表 (item)

发布时写入的核心字段：

| 字段名 | 数据类型 | 默认值 | 说明 |
|:-------|:---------|:-------|:-----|
| `id` | BIGINT | AUTO_INC | 帖文ID |
| `user_id` | BIGINT | — | 发布者ID（JWT 解析） |

| `created_at` | DATETIME | NOW() | 发布时间 |
| `expire_at` | DATETIME | NOW()+14d | 过期时间 |

### 4.3 Redis 数据结构设计

| Key 格式 | 类型 | 用途 | 过期时间 |
|:---------|:-----|:-----|:---------|
| `dict:categories` | String(JSON) | 分类字典缓存 | 1小时 |
| `upload:limit:{userId}` | String | 上传频率限制 | 60秒 |
| `sensitive:words` | Set | 违规词库（热更新用） | 永久 |

## 5. 安全设计

### 5.1 认证授权
- 所有接口均需 JWT 认证
- 发布时自动绑定当前用户 ID，防止伪造 `user_id`

### 5.2 内容安全
- **违规词库过滤**：Trie 树实现 O(n) 时间复杂度匹配
- **字数硬限制**：数据库 VARCHAR(500) 兜底，应用层 ≤200 字
- **图片格式白名单**：仅 jpg/jpeg/png/webp
- **图片大小上限**：5MB

### 5.3 风控集成
- 发布接口前置检查用户信用分 ≥ 60
- 图片上传频率限制（60秒间隔）
- 同用户发布频率限制（可选，通过 Redis 计数实现）

### 5.4 防绕过设计
- `user_id` 不从前端接收，强制从 JWT Token 解析
- `item_status` 不从前端接收，强制写死为 0（展示中）
- `created_at` / `expire_at` 不从前端接收，由服务端自动计算

## 6. 异常处理设计

### 6.1 统一异常处理
复用现有 `GlobalExceptionHandler`：

- `BusinessException`: 业务异常（分类不存在、违规内容拦截）
- `ValidationException`: 参数校验异常（字数超限、必填字段为空）
- `AuthenticationException`: 认证异常（无 Token）
- `MaxUploadSizeExceededException`: 文件过大异常（Spring 内置）

### 6.2 错误码设计

```java
CREDIT_SCORE_INSUFFICIENT(40301, "信用分不足，无法发布"),
CATEGORY_NOT_FOUND(40404, "分类不存在"),

UPLOAD_RATE_LIMITED(42902, "上传过于频繁，请60秒后再试"),
```

## 7. 性能优化设计

### 7.1 数据库优化
- 分类字典缓存至 Redis，减少数据库查询
- 发布接口单表插入，无联表操作，响应时间 < 50ms

### 7.2 图片优化
- 服务端压缩至 WebP（1080px），相比原图节省 60-80% 带宽
- 图片存储路径按年月分目录，防止单目录文件过多
- 前端上传前预压缩（可选，后续优化）

### 7.3 违规词库优化
- Trie 树内存匹配，O(n) 复杂度，200 字文本匹配耗时 < 1ms
- 支持 Redis 热更新词库，无需重启服务
