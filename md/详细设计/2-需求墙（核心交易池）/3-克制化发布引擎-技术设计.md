# 克制化发布引擎 - 后端技术设计文档

## 1. 概述

降低发布门槛，强制信息结构化。文本 ≤200字、最多1张图、必选[分类]+[校区]+[价格区间]。


## 2. 接口

### 2.1 配置字典 `GET /api/config/dict`

返回分类列表 + 校区列表 + 价格区间列表。

**校区**（硬编码）：南校区、北校区、东校区、西校区、本部

**价格区间**（硬编码，4档）：50元以下、50-200元、200元以上、面议

### 2.2 分类列表 `GET /api/config/categories`

返回 `List<CategoryResp>`，按 sort_order 排序。

### 2.3 图片上传 `POST /api/upload/image`

multipart/form-data，MIME 校验 `image/*`，≤5MB。保存至本地 `uploads/` 目录。


### 2.4 发布帖文 `POST /api/item/publish`

| 字段 | 约束 |
|------|------|
| type | 1=求购, 2=转让 |
| categoryId | 非空 |
| campus | 非空, ≤50字 |
| content | 1-200字 |
| price | ≥0（选填） |
| imageUrl | ≤255字（选填） |

逻辑：CreditInterceptor 拦截信用分<60 → 分类存在校验 → 字数校验 → 写入item表，`expire_at = now + 14天`
