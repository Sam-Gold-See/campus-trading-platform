### 长连接与实时收发引擎

**业务目标**：确保消息低延迟、不丢包，并感知对方在线状态。

#### 前端需求 (FE)
*   **WS生命周期管理**：
    *   用户登录后建立 WebSocket 连接。
    *   实现心跳机制（Ping-Pong，每30s按需发送），若断开自动实施指数退避重连。
*   **消息实时上屏**：收到推送事件立刻追加到聊天窗口，并触发滚动条触底（平滑滚动）。支持区分“己方发送（右）”与“对方发送（左）”。
*   **在线状态展示**：根据 WS 下发的状态包，在聊天页顶部昵称旁展示“当前在线/离线”的小绿点。

#### 后端需求 (BE)
*   **WS 服务端网关**：
    *   **鉴权拦截**：连接握手阶段通过 Header 或 URL 参数校验 JWT Token。
    *   **连接池管理**：维护 `User_ID -> WebSocket Channel` 的全局映射表（若多节点部署，需借助 Redis Pub/Sub 通信）。
*   **消息转发与落库逻辑**：
    *   接收到文字消息包（包含 `session_id`, `receiver_id`, `content`）。
    *   **双写策略**：先将消息异步落盘至 DB（存放 `message` 表，状态默认未读），随后检查接收方是否在线。若在线，直接通过 WS Channel 投递。
*   **状态广播**：用户上下线时，向其拥有活跃会话（最近7天有聊天）的在线列表广播状态变更通知。
