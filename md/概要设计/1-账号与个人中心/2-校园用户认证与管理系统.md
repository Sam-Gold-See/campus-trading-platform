# 产品需求文档：校园用户认证与管理系统

### 子模块 1：用户注册（`/api/user/register`）
**目标**：新用户资格校验及初始化。

#### 1.1 前端需求 (FE)
*   **邮箱校验**：输入框 Regex 实时校验。非 `@*.edu.cn` 后缀时，置灰“获取验证码”按钮，气泡提示：“仅限高校教育邮箱注册”。
*   **验证码交互**：
    *   按钮状态机：`获取验证码` -> `倒计时 (60s)` -> `重新获取`。
    *   后端返回“邮箱已注册”时，主动弹出 Toast “该邮箱已占用”，并提供“去登录”跳转。
*   **注册表单**：
    1.  校验验证码通过后，进入“完善资料”页。
    2.  需设置：**昵称**（4-16 位，禁特殊字符）、**密码**（8-20 位，需包含字母和数字）、**确认密码**。

#### 1.2 后端需求 (BE)
*   **API: 发送注册验证码** (`POST /api/user/sendRegisterCode`)
    *   **准入校验**：查询 `users` 表，若邮箱已存在，返回重复错误信息。
    *   **限流策略**：Redis 存储 `register:email -> code`，5分钟有效。
*   **API: 注册提交** (`POST /api/user/register`)
    *   **幂等与一致性**：校验验证码有效性 -> 校验昵称是否唯一 -> 写入 DB。
    *   **安全处理**：密码禁止明文，必须使用 `BCrypt` 或 `Argon2` 进行加盐哈希。
    *   **初始赋能**：`credit_score = 100`，`status = "active"`。

### 子模块 2：用户登录（`/api/user/login`）
**目标**：提供灵活的登录入口，保障访问安全。

#### 2.1 前端需求 (FE)
*   **双模式切换**：
    *   **密码登录**：邮箱 + 密码。
    *   **验证码登录**：针对忘记密码用户的快捷入口。
*   **安全防御**：连续 3 次登录失败（密码错误），前端显示图形验证码（CAPTCHA），防止暴力撞库。
*   **凭证存储**：登录成功后，将 `AccessToken` 存入 `localStorage`

#### 2.2 后端需求 (BE)
*   **API: 登录认证** (`POST /api/user/login`)
    *   **逻辑分发**：根据传参字段（`password` 或 `code`）执行不同的验证策略。
    *   **身份下发**：颁发**双 Token** 机制。
        *   `Access Token` (1d 过期)：用于 API 请求鉴权。
    *   **异常拦截**：同账号 15 分钟内连续 5 次登录失败，Redis 锁定该账号 30 分钟。

### 子模块 3：账户资料管理（`/api/user/profile`）
**目标**：支持个性化设置与账号安全加固。

#### 3.1 昵称修改 (`PUT /api/user/editNickname`)
*   **FE**：输入框字数限制，提交前进行简单的前端脱敏校验。
*   **BE**：
    *   **唯一性校验**：数据库查询，防止昵称重复。
    *   **冷却期策略**：限制每 30 天仅能修改一次，防止恶意营销脚本频繁更改账号特征。
    *   **内容审计**：接入敏感词库，拦截色情、政治、广告等违规昵称。

#### 3.2 密码修改 (`PUT /api/user/editPassword`)
*   **FE**：采用“旧密码 + 新密码 + 确认新密码”模式，包含眼睛图标（可见性切换）。
*   **BE**：
    *   **身份二次验证**：必须从 JWT 中解析当前 UID，并验证旧密码正确性。
    *   **强制下线**：改密成功后，使当前账号除本设备外的所有 Token 失效。

### 子模块 4：登出与认证失效 (`/api/user/logout`)
**目标**：闭环用户会话，防止凭证滥用。

#### 4.1 前端需求 (FE)
*   **清理逻辑**：点击“退出登录” -> 调用登出 API -> 清空本地 `UserInfo` 及 `Tokens` -> 跳转至登录页。
*   **全局拦截**：Axios 响应拦截器捕捉到 `401 Unauthorized` 时，自动执行登出逻辑。

#### 4.2 后端需求 (BE)
*   **API: 登出销毁** (`POST /api/user/logout`)
    *   **Token 黑名单**：将当前 Access Token 写入 Redis 黑名单，有效时间设为 Token 剩余过期时间。
    *   **会话管理**：清理 Redis 中该用户的 Refresh Token 记录。
