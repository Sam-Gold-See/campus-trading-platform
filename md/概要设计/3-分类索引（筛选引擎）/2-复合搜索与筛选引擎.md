### 复合搜索与筛选引擎

**业务目标**：提供灵活的多维度信息检索能力，让用户能在庞杂的信息流中迅速定位目标需求，提高撮合效率。

#### 前端需求 (FE)
*   **全局搜索框交互**：
    *   顶部提供醒目搜索栏，支持输入关键词模糊搜索。输入时支持一键清除（X按钮）。
    *   搜索触发：用户点击“搜索”或键盘回车发起请求。
*   **多维筛选面板组件**：
    *   提供横向滑动的胶囊按钮收起或下拉菜单展开筛选条件，包括：[交易类型]、[分类]、[校区]、[价格区间]。
    *   **联动刷新**：任意更改筛选条件或输入关键词，下方列表实时刷新（带局部加载骨架屏）。
    *   **空状态展示**：当搜索/筛选结果为 0 时，展示“暂无匹配内容的缺省页图案”，并引导用户“发布相关求购/转让”。
*   **状态保持（加分体验）**：
    *   将当前的筛选参数和关键词同步映射在 URL 参数上（如 `?keyword=高数&campus=南校区`），方便用户刷新不丢失进度，或分享特定筛选结果给同学。

#### 后端需求 (BE)
*   **API: 综合信息检索** (`GET /api/item/search`)
    *   **入参（全部非必填项）**：
        *   `keyword`: String (搜索关键词)
        *   `type`: Integer (1求购, 2转让，不传则查所有)
        *   `category_id`: Integer (分类ID)
        *   `campus`: String (校区标识)
        *   `price_range`: String (价格区间枚举标识)
        *   `page`, `size`: 分页参数
    *   **查询构建逻辑 (Dynamic SQL)**：
        *   **基础池限制**：必须强制带入 `status = 0` (仅搜展示中的有效帖文) 且 `expire_at > NOW()` 的条件。
        *   **多维并集拼接**：对传入的 `type`, `category_id`, `campus` 等结构化字段构建 `AND` 条件，能直接命中单列或联合索引。
        *   **全文模糊搜索实现**：如果是小并发初期，可直接使用 `content LIKE '%keyword%'`；为了长远性能，基于 MySQL 5.7+，针对 `keyword` 构建 `MATCH(content) AGAINST(? IN BOOLEAN MODE)` 语句，利用分词引擎进行高效的自然语言检索。
    *   **排序降级策略**：
        *   如果入参带着 `keyword`，优先按**文本匹配相关度得分**降序排列。
        *   如果没有 `keyword`，只是多维筛选，则按**最新时间/擦亮时间**降序排列。
    *   **防爆破限制**：针对前端传来的 `keyword` 长度进行限制（如不超过 20 字），防止过长字符串耗尽数据库检索性能。
