### 生命周期自动化管理（后端基建）

**业务目标**：避免“僵尸信息”长期占据首页视图，保持平台信息的时效性和真实性。

#### 前端需求 (FE)
*   无直接用户界面。用户仅在“我的发布”列表中能看到帖子状态变为非高亮的“已失效”。

#### 后端需求 (BE)
*   **定时任务 (Scheduled Task)**：
    *   设定机制：部署一个定时任务（Cron Job），建议低峰期每小时/每天凌晨执行一次。
    *   执行逻辑：执行 SQL 将过期帖文自动下架。
        `UPDATE item SET status = 2 WHERE status = 0 AND expire_at <= NOW()`
    *   **长条优化（可选）**：如果数据量较大，考虑分批更新，避免锁表影响正常业务读写。
